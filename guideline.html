コーディングガイドライン

いつもコーディングする上で意識しているルールです。

  • ディレクトリ階層
  • コーディングチェック
  • 命名規則
  • 複数言語共通ルール
  • HTML記述ルール
  • CSS記述ルール
  • JavaSript記述ルール

  • ディレクトリ階層

    • index.html
    •  css( cssのファイル )
    •  js( JavaScriptのファイル )
    •  img( png,jpg,svg等の画像ファイル )
    •  src( sassのファイル )
    • Sassのファイル

  • コーディングチェック

    • PCクロスブラウザチェック

      Chrome、Internet Explorer、Firefox、Safari

    • スマートフォンチェック

      Android( Chrom )、iPhone( Safari )

    • コーディングチェック

      W3Cのチェックサービスを使います。

      • HTML ⇒ W3C Markup Validation Service
      • CSS ⇒ W3C CSS Validation Service

  • 命名規則

    • ファイル名

      英単語ひと単語で命名する。下層ページはハイフンで繋げる。例：search.html search-result.html
      画像の場合は、「種類-名前00.png」のような形にする。例：icon-search01.png

    • クラス名

      • header ⇒ .head
      • main ⇒ .main
      • footer ⇒ .foot
      • sectionをまとめるdiv ⇒ .contents
      • section ⇒ .content-〇〇 例：.content-search
      • sectionの中の部品 ⇒ .〇〇-△△ 例：.search-head

    • 変数名

      javaScript等で使用する変数名は、クラス名を参考してキャメルケースで命名する。例：var searchHead

  • 複数言語共通ルール

    • 1行1要素(HTML,CSS)

      なるべく1行に1要素しか書かないようにする。
      HTMLの閉じタグが不要なタグや、1部のインライン要素などはOK。
      例：<p>これは<span>サンプル</span>です。</p>

    • レスポンシブ対応(HTML,CSS)

      レスポンシブ対応させることを意識してコーディングする。@mediaを使用し、適切な見た目になるようにする。
      一番見る人が多いであろうデバイスから作っていく。

    • コメント(HTML,CSS,JS)

      • HTML ⇒ <!-- コメント -->
      • CSS ⇒ /* コメント */
      • JavaScript ⇒ // コメント

    • クォーテーション

      JavaScript等で特別な場合を除いて、基本的にはダブルクォーテーション("")を使用する。

  • HTML記述ルール

    • doctype宣言

      HTML5を使用する。⇒<!DOCTYPE html>

    • 使用言語の指定

      日本語⇒lang="ja"

    • metaタグの記述

      • <meta charset="UTF-8">
      • <meta name="viewport" content="width=device-width, initial-scale=1.0">
      • <meta http-equiv="X-UA-Compatible" content="ie=edge">
      • <meta name="description" content="ページの説明">

    • タイトルタグの記述

      <title>タイトル</title>

    • タグについて

      • HTML5のタグを使用する。例：header,main,section
      • h1はロゴなど見出しに使う。
      • 線や丸などは、spanタグを使用する。
      • aタグやimgタグは、pタグで囲む。

    • 属性について

      • alt属性は必ず書く。iconなどの見た目で使用している、意味を持たないものは、不要。
      • 記述順は、id ⇒ class ⇒ 他属性。styleは書かない

    • インデント

      tabを使用し、揃える。

    • 特殊文字の記述

      • " ⇒ &quot;
      • & ⇒ &amp;
      • < ⇒ &lt;
      • > ⇒ &gt;
      • 半角スペース ⇒ &nbsp;
      • ほぼ全角スペース ⇒ &emsp;
      • © ⇒ &copy;

  • CSS記述ルール

    • Sassを使用する。

      そのページでよく使用するフォントや色は、変数に登録する。

    • ressファイルを使用する。

    • 共通スタイル

      複数のページで繰り返し使用するスタイルは、_common.cssに記述し、@importで読み込む。

    • 繰り返し使うスタイルはクラス名を作る。

      例：display:flex ⇒ .flexというクラスを作り、HTMLのタグに.flexを記述。

    • プロパティの記述順

      表示系⇒トランジション⇒サイズ系⇒位置系⇒その他スタイル
      display(flex),float⇒opacity⇒overflow⇒transition
      ⇒width⇒height⇒margin⇒padding⇒border
      ⇒position⇒transform⇒その他

    • 単位

      • htmlには62.5%を指定する。
      • line-height:なし
      • border,border-radius:px
      • letter-spacing,font-size:em,rem

    • 省略するもの・まとめるもの

      • margin・padding
      • background
      • カラーコード3桁にできるものは省略。(小文字)

  • JavaScript記述ルール

    • プラグインは使わない。

      参考にするのは可だが、自分でコーディングする。

    • j-Queryの使用

      バージョン3.3.1のファイルを使用。

    • cssスタイル

      クラス名.onなどをタグにつけて、それを追加・削除することで変化させる。
